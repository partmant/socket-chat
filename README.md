## 프로젝트 개요

이 프로젝트는 WebSocket을 활용한 실시간 채팅 애플리케이션입니다.  
사용자는 채팅방을 생성하거나 목록에서 방을 선택하여 입장할 수 있으며,  
다른 사용자들과 실시간으로 메시지를 주고받을 수 있습니다.  
비밀번호 방, 인원 제한, 방 삭제 등 기본적인 채팅방 관리 기능을 구현하는 것을 목표로 합니다.

## 구현할 기능 목록

### 방 목록 조회
- [x] 현재 생성된 방 목록을 조회한다.
  - 방 제목, 비밀번호 여부를 포함한다. (현재 인원 수는 추후 구현)
- [x] 사용자가 검색어를 전달하면 제목에 검색어를 포함한 방만 조회한다.
  - 검색어가 null 또는 공백이면 전체 목록을 조회한다.
  - 검색 결과가 없으면 빈 목록을 반환한다.

### 방 생성
- [x] 사용자가 방 생성 요청을 보낸다.
  - [x] 방 제목을 입력받는다.
    - null 또는 공백이면 예외를 발생시킨다.
    - 1~50자가 아니면 예외를 발생시킨다.
    - 이미 존재하는 제목이면 예외를 발생시킨다.
  - [x] 비밀번호를 설정할 수 있다.
    - 입력하지 않으면 비밀번호 없는 방으로 생성된다. 
    - 4~10자가 아니면 예외를 발생시킨다.
  - [x] 방 허용 인원을 입력받는다.
    -  허용 인원이 1~10 이내가 아니면 예외를 발생시킨다.

### 방 입장
- [ ] 사용자가 방 입장 요청을 보낸다.
  - 닉네임을 입력받는다.
    - 닉네임이 1~16자가 아니면 예외를 발생시킨다.
    - 닉네임이 해당 방 내에서 중복되면 예외를 발생시킨다.
  - 비밀번호가 있는 방은 비밀번호를 입력받는다.
    - 비밀번호가 일치하지 않으면 예외를 발생시킨다.
- [ ] 방의 현재 인원이 10명을 초과하면 입장을 허용하지 않는다.
- [ ] 입장 성공 시 입장 메시지를 같은 방 사용자에게 전송한다.

### 방 퇴장
- [ ] 사용자가 방에서 나가면 퇴장 메시지를 같은 방 사용자에게 전송한다.

### 방 삭제
- [ ] 방을 생성한 사용자가 방에서 나가면 해당 방을 삭제한다.
  - [ ] 방 삭제 시 모든 사용자에게 방 삭제 메시지를 전송한다.
  - [ ] 사용자를 자동으로 방 목록 화면으로 이동시킨다.

### 채팅 메시지 송수신
- [ ] 사용자가 메시지를 전송한다.
  - 메시지가 200자 이상이면 예외를 발생시킨다.
- [ ] 서버는 방에 속한 모든 사용자에게 메시지를 전송한다.
